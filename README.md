# i12 Cloud Broker

Independent cloud phone system broker for small businesses.

I help you compare providers, choose the right cloud phone system, and avoid bad setups. Everything is explained in plain English.

## What i12 Cloud Broker does

Typical projects include:

- Cloud phone system selection (feature fit, pricing, and tradeoffs)
- Provider comparisons and recommendations
- Call flow planning and setup
	- IVR / auto-attendant menus
	- Queues and routing
	- Business hours and after-hours routing
	- Voicemail strategy
- Number porting coordination and cutover planning
- Fix-and-optimize audits for messy or unreliable systems

---

## Why clients choose this approach

Common feedback themes:

- **Clarity:** Simple explanations and documented decisions.
- **Ownership:** Follow-through until calls and routing work correctly.
- **Problem-solving:** Creative solutions when the “default” setup does not fit.

## Publishing (GitHub Pages)

1. Repo → **Settings → Pages**
2. **Source:** Deploy from a branch
3. **Branch:** `main` and `/ (root)`
4. Save and wait a few minutes for the site to publish.

## Custom domain (DNS)

For `www.i12cloudbroker.com`:

- In GitHub Pages, set the **Custom domain** to: `www.i12cloudbroker.com`
- In your domain registrar DNS, set:
	- **CNAME**: `www` → `i12connect.github.io`

Recommended:

- Redirect `i12cloudbroker.com` → `www.i12cloudbroker.com` at your registrar.
